# Каталог формул Excel-отчета

Документ фиксирует ключевую расчетную логику файла `result/data_quality_report.xlsx`.

## Проверочные листы

| Лист | Назначение |
|---|---|
| `01_orders_raw` | исходная выгрузка заказов |
| `02_clients_raw` | исходная выгрузка клиентов |
| `03_payments_raw` | исходная выгрузка оплат |
| `04_checks` | формульные проверки каждой записи |
| `05_errors` | бизнес-реестр найденных событий ошибок |
| `06_summary` | сводные формулы для руководителя |
| `07_dashboard` | дашборд на основе расчетных показателей |
| `08_action_plan` | план исправления ошибок |

## Основные проверки

| Проверка | Пример формульной логики |
|---|---|
| Дубли заказов | `COUNTIF(order_id_range, order_id) > 1` |
| Пустые обязательные поля | `IF(OR(client_id="", manager=""), "ERROR", "OK")` |
| Связь заказа с клиентом | `COUNTIF(client_id_range, order.client_id) = 0` |
| Корректность суммы заказа | `order_sum <> quantity * price` |
| Отрицательная цена | `price < 0` |
| Будущая дата заказа | `order_date > DATE(2026,6,25)` |
| Статус по справочнику | проверка значения через список допустимых статусов |
| Оплата связана с заказом | `COUNTIF(order_id_range, payment.order_id) = 0` |
| Оплата не превышает заказ | `payment_sum > matched_order_sum` |
| Дата оплаты не раньше заказа | `payment_date < matched_order_date` |

## Сводные показатели

- количество заказов, клиентов и оплат считается через `COUNTA`;
- сумма заказов считается через `SUM`;
- количество ошибок считается по реестру `05_errors`;
- доля записей с ошибками считается как доля строк `04_checks`, где `error_count > 0`;
- распределение ошибок по типам считается через `COUNTIF`;
- динамика заказов по месяцам считается через `SUMIFS`;
- показатели дашборда ссылаются на лист `06_summary`.

## Зачем это нужно

Формульная структура делает отчет проверяемым: пользователь может изменить исходную выгрузку на raw-листах и увидеть, какие проверки сработают на листе `04_checks`, а затем как изменятся сводка и дашборд.
