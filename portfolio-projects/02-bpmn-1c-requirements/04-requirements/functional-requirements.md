# Functional Requirements

| ID | Requirement | Priority |
|---|---|---|
| FR-01 | Создать документ 1С `Заявка` с единой формой ввода | High |
| FR-02 | Сделать обязательными поля: инициатор, тема, описание, категория, приоритет | High |
| FR-03 | Запретить сохранение заявки без обязательных полей | High |
| FR-04 | Использовать справочник статусов вместо ручного ввода | High |
| FR-05 | Автоматически рассчитывать срок обработки по приоритету | High |
| FR-06 | Назначать ответственного менеджера при создании заявки | High |
| FR-07 | Отправлять уведомление ответственному при создании заявки | Medium |
| FR-08 | Отправлять уведомление руководителю при просрочке SLA | High |
| FR-09 | Переводить заявки высокого приоритета на согласование | Medium |
| FR-10 | Сохранять историю изменения статусов | Medium |
| FR-11 | Добавить отчет руководителя по статусам, SLA и ответственным | High |
| FR-12 | Ограничить доступ к изменению справочников и статусов | Medium |

## Status Model

| Status | Description |
|---|---|
| Новая | Заявка создана и ожидает проверки |
| Требует уточнения | Не хватает обязательной информации |
| В работе | Менеджер принял заявку |
| На согласовании | Ожидается решение руководителя |
| Согласована | Можно передавать в выполнение |
| Отклонена | Заявка не принята |
| Выполнена | Работы завершены |
| Закрыта | Заявка проверена и закрыта |
