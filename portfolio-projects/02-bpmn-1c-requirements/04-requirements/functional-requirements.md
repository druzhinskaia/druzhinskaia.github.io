# Функциональные требования

| ID | Требование | Приоритет |
|---|---|---|
| FR-01 | Создать документ 1С `Заявка` с единой формой ввода | Высокий |
| FR-02 | Сделать обязательными поля: инициатор, тема, описание, категория, приоритет | Высокий |
| FR-03 | Запретить сохранение заявки без обязательных полей | Высокий |
| FR-04 | Использовать справочник статусов вместо ручного ввода | Высокий |
| FR-05 | Автоматически рассчитывать контрольный срок обработки по приоритету | Высокий |
| FR-06 | Назначать ответственного менеджера при создании заявки | Высокий |
| FR-07 | Отправлять уведомление ответственному при создании заявки | Средний |
| FR-08 | Отправлять уведомление руководителю при нарушении контрольного срока | Высокий |
| FR-09 | Переводить заявки высокого приоритета на согласование | Средний |
| FR-10 | Сохранять историю изменения статусов | Средний |
| FR-11 | Добавить отчет руководителя по статусам, контрольным срокам и ответственным | Высокий |
| FR-12 | Ограничить доступ к изменению справочников и статусов | Средний |

## Модель статусов

| Статус | Описание |
|---|---|
| Новая | Заявка создана и ожидает проверки |
| Требует уточнения | Не хватает обязательной информации |
| В работе | Менеджер принял заявку |
| На согласовании | Ожидается решение руководителя |
| Согласована | Можно передавать в выполнение |
| Отклонена | Заявка не принята |
| Выполнена | Работы завершены |
| Закрыта | Заявка проверена и закрыта |
