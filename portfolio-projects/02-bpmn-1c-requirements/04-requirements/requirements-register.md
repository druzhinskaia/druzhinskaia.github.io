# Requirements Register

| ID | Требование | Приоритет | Роль | Критерий приемки |
|---|---|---|---|---|
| FR-01 | Создать единую форму заявки в 1С | High | Менеджер | Пользователь может создать заявку через форму 1С |
| FR-02 | Сделать обязательными ключевые поля | High | Менеджер | Система не сохраняет заявку без обязательных полей |
| FR-03 | Запретить сохранение заявки без обязательных полей | High | Менеджер | Система показывает ошибку валидации |
| FR-04 | Использовать справочник статусов вместо ручного ввода | High | Менеджер | Статус выбирается из утвержденного списка |
| FR-05 | Рассчитывать срок обработки по приоритету | High | 1С | После выбора приоритета автоматически заполняется SLA |
| FR-06 | Назначать ответственного менеджера | High | 1С | У заявки всегда есть ответственный |
| FR-07 | Отправлять уведомление ответственному | Medium | Менеджер | При создании заявки ответственный получает уведомление |
| FR-08 | Уведомлять руководителя о просрочке | High | Руководитель | При нарушении срока руководитель получает уведомление |
| FR-09 | Добавить согласование нестандартных заявок | Medium | Руководитель | Высокий приоритет переводит заявку на согласование |
| FR-10 | Сохранять историю изменения статусов | Medium | Аналитик | В карточке заявки видны дата, пользователь и новый статус |
| FR-11 | Добавить отчет руководителя | High | Руководитель | Отчет показывает статусы, просрочки и ответственных |
| FR-12 | Ограничить доступ к справочникам | Medium | Администратор | Изменять справочники может только администратор |

## Приоритеты

- `High` - влияет на управляемость процесса и контроль сроков.
- `Medium` - повышает удобство и прозрачность.
- `Low` - может быть реализовано после запуска основной версии.
