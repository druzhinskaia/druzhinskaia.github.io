# Реестр требований

| ID | Требование | Приоритет | Роль | Критерий приемки |
|---|---|---|---|---|
| FR-01 | Создать единую форму заявки в 1С | Высокий | Менеджер | Пользователь может создать заявку через форму 1С |
| FR-02 | Сделать обязательными ключевые поля | Высокий | Менеджер | Система не сохраняет заявку без обязательных полей |
| FR-03 | Запретить сохранение заявки без обязательных полей | Высокий | Менеджер | Система показывает ошибку валидации |
| FR-04 | Использовать справочник статусов вместо ручного ввода | Высокий | Менеджер | Статус выбирается из утвержденного списка |
| FR-05 | Рассчитывать контрольный срок обработки по приоритету | Высокий | 1С | После выбора приоритета автоматически заполняется контрольный срок |
| FR-06 | Назначать ответственного менеджера | Высокий | 1С | У заявки всегда есть ответственный |
| FR-07 | Отправлять уведомление ответственному | Средний | Менеджер | При создании заявки ответственный получает уведомление |
| FR-08 | Уведомлять руководителя о просрочке | Высокий | Руководитель | При нарушении контрольного срока руководитель получает уведомление |
| FR-09 | Добавить согласование нестандартных заявок | Средний | Руководитель | Высокий приоритет переводит заявку на согласование |
| FR-10 | Сохранять историю изменения статусов | Средний | Аналитик | В карточке заявки видны дата, пользователь и новый статус |
| FR-11 | Добавить отчет руководителя | Высокий | Руководитель | Отчет показывает статусы, просрочки и ответственных |
| FR-12 | Ограничить доступ к справочникам | Средний | Администратор | Изменять справочники может только администратор |

## Приоритеты

- `Высокий` - влияет на управляемость процесса и контроль сроков.
- `Средний` - повышает удобство и прозрачность.
- `Низкий` - может быть реализован после запуска основной версии.
