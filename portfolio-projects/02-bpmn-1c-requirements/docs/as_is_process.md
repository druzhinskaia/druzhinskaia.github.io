# AS-IS процесс обработки заявки

## Краткое описание

Текущий процесс построен вокруг ручной обработки. Заявка поступает менеджеру, проверяется вручную, затем данные переносятся в 1С. Если данных не хватает, менеджер запрашивает уточнения. Согласование и контроль сроков происходят вне системы.

## Участники

- Клиент или инициатор.
- Менеджер.
- Руководитель.
- 1С.
- Исполнитель.

## Шаги процесса

1. Клиент отправляет заявку по почте или в мессенджере.
2. Менеджер вручную проверяет состав заявки.
3. Если данных не хватает, менеджер запрашивает уточнение.
4. После получения данных менеджер вручную создает документ в 1С.
5. При необходимости менеджер отправляет заявку руководителю на согласование.
6. Руководитель согласует или возвращает заявку на доработку.
7. Менеджер передает заявку исполнителю.
8. После выполнения менеджер вручную меняет статус заявки.
9. Руководитель получает информацию о ходе процесса только по запросу.

## Основные ограничения

- Нет единого входного канала заявок.
- Нет обязательных полей на этапе создания заявки.
- Статусы вводятся вручную и могут отличаться.
- Нет автоматического контроля сроков.
- Нет уведомлений о просрочках.
- Руководитель не видит полную картину без ручной сверки.
