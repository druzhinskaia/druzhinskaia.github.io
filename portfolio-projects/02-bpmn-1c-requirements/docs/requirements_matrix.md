# Матрица требований

| ID | Требование | Приоритет | Роль | Критерий приемки |
|---|---|---|---|---|
| FR-01 | Создать единую форму заявки в 1С | High | Менеджер | Пользователь может создать заявку через форму 1С |
| FR-02 | Сделать обязательными ключевые поля | High | Менеджер | Система не сохраняет заявку без обязательных полей |
| FR-03 | Использовать справочник статусов | High | Менеджер | Статус выбирается из списка, ручной ввод невозможен |
| FR-04 | Рассчитывать срок обработки по приоритету | High | 1С | После выбора приоритета автоматически заполняется срок |
| FR-05 | Отправлять уведомление ответственному | Medium | Менеджер | При создании заявки ответственный получает уведомление |
| FR-06 | Уведомлять руководителя о просрочке | High | Руководитель | При нарушении срока руководитель получает уведомление |
| FR-07 | Добавить согласование нестандартных заявок | Medium | Руководитель | Высокий приоритет переводит заявку на согласование |
| FR-08 | Сохранять историю изменения статусов | Medium | Аналитик | В карточке заявки видны дата, пользователь и новый статус |
| FR-09 | Добавить отчет руководителя | High | Руководитель | Отчет показывает статусы, просрочки и ответственных |
| FR-10 | Ограничить доступ к справочникам | Medium | Администратор | Изменять справочники может только администратор |

## Приоритеты

- `High` - влияет на управляемость процесса и контроль сроков.
- `Medium` - повышает удобство и прозрачность.
- `Low` - может быть реализовано после запуска основной версии.
