# Project Summary

## Case

Смоделированный коммерческий кейс по автоматизации обработки клиентских заявок в 1С для компании ООО "ТехСнаб".

## Business Problem

Заявки поступают из разных каналов, обрабатываются вручную и не имеют единого контроля сроков. Из-за этого возникают просрочки, неполные данные, разные статусы и ручная отчетность.

## Analyst Work

- описан бизнес-контекст и stakeholders;
- описаны goals, KPI, risks, assumptions и constraints;
- собраны пользовательские проблемы через интервью;
- описан AS-IS процесс и подготовлена BPMN-схема;
- проведен root cause analysis;
- сформирован scope проекта;
- подготовлены business, functional и non-functional requirements;
- подготовлена requirements traceability matrix;
- спроектирован TO-BE процесс;
- добавлены architecture, use case и sequence diagrams;
- подготовлен UI mockup формы заявки;
- составлено техническое задание на доработку 1С;
- описаны acceptance criteria и тестовые сценарии.

## Deliverables

- AS-IS BPMN;
- TO-BE BPMN;
- Business Requirements;
- Functional Requirements;
- Non-Functional Requirements;
- User Stories;
- Requirements Traceability Matrix;
- Technical Specification;
- UI Mockup;
- Acceptance Criteria;
- Test Scenarios;
- Demo Outline.

## Result

Подготовлен пакет аналитических артефактов, который можно передать команде 1С для оценки и реализации доработки: от бизнес-проблемы и KPI до требований, схем, UI mockup, ТЗ и приемочных сценариев.

## Skills

BPMN, draw.io, business analysis, process analysis, root cause analysis, requirements management, RTM, user stories, technical specification, acceptance criteria, 1C.
